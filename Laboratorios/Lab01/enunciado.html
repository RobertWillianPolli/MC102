Aritmética com Inteiros

O objetivo deste laboratório é exercitar o processo de submissão de um código em Python no sistema SuSy. Seu programa irá receber três números inteiros e irá armazená-los nas variáveis a, b e c. Como saída, seu programa deve exibir inicialmente os valores das variáveis a, b e c. Em seguida, seu programa deve efetuar uma sequência de operações aritméticas utilizando as variáveis a, b e c e exibir o resultado na seguinte ordem:

  • Soma a + b + c
  • Subtração a - b - c
  • Multiplicação a * b * c
  • Exponenciação a ** c
  • Divisão Inteira b // c
  • Resto da Divisão b % c

Exemplos de entradas e saídas esperadas pelo seu programa:

Código Base

No arquivo auxiliar lab01.py você irá encontrar um código base para dar início ao processo de submissão deste laboratório. Você deve realizar a primeira submissão no SuSy utilizando esse código base e irá perceber que em todos os casos de testes as soluções serão consideradas incorretas. No arquivo auxiliar lab01.py existem três erros que devem ser corrigidos para que a submissão seja considerada correta em todos os casos de teste. A seguir iremos mostrar como corrigir esses erros presentes no código base. Para cada um dos erros você deve realizar uma nova submissão no SuSy para verificar se o erro foi corrigido ou não.

Corrigindo o Código Base

1. Espaçamento Extra

Na linha 13 do código base você irá encontrar o seguinte comando:

print("b  =", b)

Perceba que antes da igualdade existem dois espaços, mas no exemplo de saída esperada pelo seu programa existe apenas um. Para cada um dos testes o SuSy compara se a saída do seu programa é exatamente igual a saída esperada do teste, isso significa que espaços extras podem fazer com que seu programa forneça soluções incorretas mesmo que visualmente elas pareçam idênticas. Para solucionar esse problema modifique a linha 13 para o seguinte comando:

print("b =", b)

Realize a submissão novamente no SuSy e verifique se o problema de espaço extra foi corrigido.

2. Ordem dos Operandos

Na linha 16 do código base você irá encontrar o seguinte comando:

print("a - b - c =", c - a - b)

Note que nesse comando é esperado que o resultado da operação a - b - c seja exibido, mas o comando está exibindo o resultado da operação c - a - b e isso torna as saídas fornecidas pelo seu programa incorretas. Para solucionar esse problema modifique a linha 16 para o seguinte comando:

print("a - b - c =", a - b - c)

Realize a submissão novamente no SuSy e verifique se o problema foi corrigido.

3. Ordem das Impressões

Nas linhas 19 e 20 do código base você irá encontrar os seguintes comandos:

print("b % c =", b % c)
print("b // c =", b // c)

Note que programa será executado linha por linha, isso significa que o código base está exibindo primeiro o resultado da operação b % c (resto da divisão inteira) antes do resultado da operação b // c (divisão inteira). Como mencionado anteriormente, o SuSy compara se a saída do seu programa é exatamente igual a saída esperada para cada um dos testes e como a exibição dos resultados das operações aritméticas está invertida, os testes serão considerados como incorretos. Para solucionar esse problema, inverta a ordem das linhas 19 e 20, desta forma:

print("b // c =", b // c)
print("b % c =", b % c)

Realize a submissão novamente no SuSy e verifique se para todos os testes as soluções constam como corretas.
